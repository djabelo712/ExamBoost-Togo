# Golden tests

## What are golden tests?

Golden tests capture a pixel-perfect screenshot of a widget and compare it
against a baseline (the "golden file"). If the rendered widget differs from
the baseline by more than a small threshold, the test fails.

They are useful for:
- Detecting visual regressions (color, spacing, typography changes).
- Documenting the expected look of each screen.

## Layout

```
test/golden/
├── home_screen_golden_test.dart   # Test driver
├── goldens/                        # Baseline images (.png)
│   ├── home_screen.png             # Generated by --update-goldens
│   ├── home_screen_no_user.png
│   ├── onboarding_welcome.png
│   └── srs_buttons.png
└── README.md                       # This file
```

## Generating baselines

Run the following command from the project root (`ExamBoost-Togo/`):

```bash
flutter test --update-goldens test/golden/home_screen_golden_test.dart
```

The `.png` files are written to `test/golden/goldens/`. Commit them to git
so the CI can compare against them.

## Verifying (CI / local)

Once the baselines are committed:

```bash
flutter test test/golden/home_screen_golden_test.dart
```

If a test fails, `flutter test` produces a diff image (`.isolatedDiff.png`)
next to the original. Inspect it to decide whether the change is intended.

## Platform-specific rendering

Golden images are platform-specific because font rendering and anti-aliasing
differ between macOS, Linux, and Windows. To get reproducible results in CI,
run the tests on the same OS as the developer who generated the baselines.

If you need cross-platform goldens, consider using `flutter test --platform=chrome`
(which renders in a headless browser) or wrap the test in a fixed-pixel
`MaterialApp` with a `ThemeData` that uses a bundled font.

## Adding a new golden test

1. Add a `testWidgets('MyWidget golden', ...)` block to a `_golden_test.dart`
   file in this directory.
2. Use `matchesGoldenFile('goldens/my_widget.png')` as the matcher.
3. Run `flutter test --update-goldens test/golden/my_widget_golden_test.dart`
   to generate the baseline.
4. Commit the `.png` file.

## Troubleshooting

- **"Golden file not found"**: run with `--update-goldens` first.
- **Test fails on CI but not locally**: the rendering differs. Check the OS
  and Flutter version. Consider regenerating the golden on the CI machine.
- **Anti-aliasing artifacts**: wrap the widget in `RepaintBoundary` and use
  `tester.binding.window.physicalSizeTestValue` to fix the resolution.
